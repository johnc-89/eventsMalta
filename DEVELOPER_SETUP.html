Local Development Setup

Steps for a new developer to get running locally:

1
Clone the repo 
git clone https://github.com/johnc-89/eventsMalta.git
cd eventsMalta
2
Create .env.local
Copy the example file and fill in the values from Supabase: 
cp .env.local.example .env.local
3
Install dependencies 
npm install
4
Start the dev server 
npm run dev
Visit http://localhost:3000

Deployment (Vercel)

The project auto-deploys to Vercel on every push to the main branch.

How it works

  1. Push code to GitHub: git push
  2. Vercel detects the push and triggers a new build
  3. If the build passes, the live site is updated automatically

Vercel environment variables

Set in Vercel Dashboard → Project → Settings → Environment Variables:

  • NEXT_PUBLIC_SUPABASE_URL
  • NEXT_PUBLIC_SUPABASE_ANON_KEY
Important: The GitHub repo must remain public for Vercel's free Hobby plan to deploy it. Upgrading to Vercel Pro ($20/month) allows private repos.

Daily Development Workflow

# Start local server
